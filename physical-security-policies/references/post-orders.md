# Security Officer Post Orders Reference

## Domain-Specific Inputs Required

- Guard force type (proprietary, contract, hybrid)
- Contract security provider (if applicable)
- Number and location of posts
- Armed or unarmed officers
- Post hours (24/7, business hours, event-based)
- Uniform and equipment standards
- Communication equipment (radio, phone, body camera)
- Patrol routes and frequency
- Access to firearms or less-lethal tools (OC spray, baton, handcuffs)
- Use of force policy (separate document or included)
- Licensing and training requirements by state

## Key Post Order Provisions to Include

### Post Order Structure
Each post shall have a dedicated post order document containing:

**Header Information:**
- Post Name and Number
- Location / Address
- Post Type (fixed, roving, dispatch, supervisor)
- Hours of Operation
- Staffing Level
- Effective Date and Version

**Post-Specific Information:**

1. **Post Overview and Purpose**
   - Why this post exists
   - Primary security objectives for this location
   - Key assets or personnel being protected

2. **Post Duties -- Routine**
   - Opening/closing procedures
   - Shift changeover procedures (briefing, equipment check, log review)
   - Access control duties (badge verification, visitor processing, vehicle screening)
   - Patrol routes and schedules (randomized timing where applicable)
   - Monitoring duties (camera feeds, alarm panels, fire panels)
   - Logging requirements (activity log entries, minimum frequency)
   - Package and delivery screening
   - Parking enforcement

3. **Post Duties -- Emergency**
   - Fire alarm response (specific to this post location)
   - Medical emergency response
   - Active threat/lockdown response
   - Bomb threat response
   - Severe weather response
   - Power outage procedures
   - Suspicious person/vehicle response
   - Cross-reference to Emergency Response Procedure for detailed protocols

4. **Communication Protocols**
   - Radio channel assignments
   - Check-in schedule with dispatch/SOC
   - Escalation contacts by priority:
     - Routine: [USER INPUT REQUIRED]
     - Urgent: [USER INPUT REQUIRED]
     - Emergency: 911, then [USER INPUT REQUIRED: security management contact]
   - Prohibited communications (personal calls, social media while on post)

5. **Access Control at Post**
   - Authorized personnel and credential requirements
   - Visitor processing specific to this post
   - Delivery and vendor access procedures
   - Denied entry procedures
   - Tailgating prevention responsibilities

6. **Use of Force** (if applicable)
   - Cross-reference to Use of Force Policy
   - Force continuum summary:
     1. Officer presence
     2. Verbal commands
     3. Soft hands (escort, guide)
     4. Hard hands (defensive)
     5. Less-lethal tools (if authorized)
     6. Lethal force (if armed -- life-threatening situations only)
   - Reporting requirements for any use of force
   - [USER INPUT REQUIRED: State-specific use of force legal framework]

7. **Report Writing**
   - Daily Activity Report (DAR) requirements
   - Incident report triggers and format
   - Report submission timeline
   - Report review process

8. **Equipment and Uniform**
   - Required uniform components
   - Required equipment (radio, flashlight, keys, phone, body camera)
   - Equipment inspection at shift start
   - Equipment malfunction reporting
   - Prohibited items

9. **Key Contacts**
   - Security Operations Center
   - Security management (on-call)
   - Facilities/maintenance
   - Local police non-emergency
   - Local fire department non-emergency
   - Building management
   - IT help desk (for access system issues)

10. **Post-Specific Hazards and Safety**
    - Known hazards at this post location
    - PPE requirements
    - Lone worker safety procedures (if applicable)
    - Weather exposure considerations

### Patrol Procedures (for Roving Posts)
- Defined patrol route with checkpoints
- Checkpoint verification method (electronic guard tour system, manual log)
- Patrol frequency: [USER INPUT REQUIRED: e.g., every 60 minutes]
- Randomization requirements (vary route, timing)
- What to inspect during patrol:
  - Doors locked/secured
  - Lighting functional
  - Fire exits clear
  - No unauthorized persons
  - No signs of forced entry or damage
  - Hazard identification
- Documentation of findings

### Shift Changeover Procedures
1. Outgoing officer briefs incoming officer on:
   - Outstanding incidents
   - Equipment status
   - Pending deliveries or visitors
   - Management instructions
   - Any anomalies observed
2. Joint equipment check
3. Log book signed by both officers
4. Keys and access devices transferred and verified
5. Radio check

## Compliance Metrics

- Post coverage rate (hours staffed vs. hours required)
- Patrol checkpoint completion rate
- Daily Activity Report submission rate and timeliness
- Incident report quality scores (completeness, accuracy)
- Officer training currency rate
- Guard tour completion rate
- Use of force incident rate and review completion

## Common Legal Considerations

- State security officer licensing requirements (armed and unarmed)
- State-specific use of force laws and civil liability
- Contract security: Ensure post orders are incorporated into or consistent with service agreement
- Workers compensation: Officers injured on duty
- Liability for officer actions (respondeat superior, negligent hiring/training/supervision)
- Body camera laws vary by state (consent, retention, disclosure)
- FLSA: Guard breaks, meal periods, and compensable time considerations
