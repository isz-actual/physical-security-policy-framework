# Key & Lock Control Policy Reference

## Domain-Specific Inputs Required

- Key system type (mechanical, restricted keyway, electronic/smart lock, combination)
- Master key system architecture (if applicable)
- Key management software or manual log system
- Number of keyed entry points
- High-security lock requirements (UL 437, ANSI/BHMA Grade 1)
- Safe and vault inventory
- Integration with electronic access control (hybrid systems)
- Key cutting performed in-house or by vendor

## Key Policy Provisions to Include

### Key Classification
Define key tiers aligned with access zones:
- **Grand Master**: Facility-wide access. Limited to [USER INPUT REQUIRED: role, e.g., Director of Security, Facilities Director]
- **Master**: Building or floor-level access
- **Sub-Master**: Department or zone-level access
- **Individual/Change Key**: Single door or space
- **Emergency/Break Glass**: Sealed, auditable emergency access keys

### Key Issuance and Tracking
- All keys assigned to named individuals (no shared keys without documented exception)
- Key receipt form signed at issuance
- Key inventory database maintained by [USER INPUT REQUIRED: responsible party]
- Unique key number stamped or engraved (no "Do Not Duplicate" reliance alone)
- Restricted keyway blanks controlled by vendor agreement

### Key Return and Recovery
- Return required upon: termination, transfer, role change, leave of absence
- Key return tracked and signed
- Failure to return triggers:
  1. Documented follow-up within [USER INPUT REQUIRED: e.g., 24 hours]
  2. If unrecovered within [USER INPUT REQUIRED: e.g., 72 hours], lock re-key assessment
  3. Cost recovery from departing employee (if permitted by policy/law)

### Lost or Stolen Keys
- Immediate reporting requirement to [USER INPUT REQUIRED: reporting chain]
- Risk assessment by security within [USER INPUT REQUIRED: e.g., 4 hours]
- Re-key decision matrix:
  - Individual key: Re-key affected lock
  - Sub-master or higher: Re-key assessment for all affected locks
  - Grand master: Mandatory system-wide re-key assessment
- Incident report filed for all lost/stolen key events

### Key Audits
- Physical key audit frequency: [USER INPUT REQUIRED: e.g., quarterly for master+, annually for all keys]
- Reconciliation of issued keys vs. physical inventory
- Audit of vendor key cutting records (if applicable)
- Safe combination change schedule: [USER INPUT REQUIRED: e.g., annually and upon personnel change]

### Safe and Vault Controls
- Combination limited to named individuals
- Combination change triggers: personnel change, compromise suspected, scheduled rotation
- Dual-control requirements for high-value storage
- Time-delay lock requirements (if applicable)
- Safe/vault inspection schedule

### Electronic Lock Integration
- Smart lock credential management follows Access Control Policy
- Battery replacement and monitoring schedule
- Audit trail review frequency
- Offline access/override procedures

## Compliance Metrics

- Key audit reconciliation rate (target: 100%)
- Average time from termination to key recovery
- Number of unreturned keys per quarter
- Re-key events triggered by lost/stolen keys per year
- Safe combination change compliance rate
- Percentage of high-security locks meeting ANSI/BHMA or UL standards

## Common Legal Considerations

- State landlord-tenant laws may govern key duplication and lock change rights (if facilities include leased space)
- Union/CBA may restrict changes to key access without notification
- Cost recovery for lost keys must comply with state wage deduction laws
- ADA: Keyed entry points must have accessible alternatives
