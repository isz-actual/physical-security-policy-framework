# Physical Security Policy Framework

**ISO-aligned, audit-ready physical security policies and procedures for corporate security programs.**

Built by a CPP-certified security practitioner with 26+ years of experience across aerospace, autonomous vehicles, high-tech, and supply chain operations. Designed to make ISO 27001 and ISO 22301 certification preparation faster and less painful.

## What This Is

A structured framework for generating professional physical security policies that are:

- **ISO-mapped** -- every policy includes a control mapping table linking sections to ISO 27001 Annex A.11 and ISO 22301 controls
- **Audit-ready** -- consistent document control structure across all policy domains with built-in revision tracking
- **Legally aware** -- jurisdiction-specific compliance flags for ADA, OSHA, BIPA, GDPR, state wiretapping laws, and union/CBA considerations
- **Gap-proof** -- unknown or site-specific variables are explicitly flagged with `[USER INPUT REQUIRED]` tags so nothing gets silently assumed

## How It Works

This framework operates as a **Claude AI skill**. When installed, it instructs Claude to:

1. Identify the policy domain from your request
2. Load the relevant reference file for that domain
3. Collect required inputs (or flag them for your review)
4. Generate a complete policy document following a consistent structure
5. Append ISO control mapping and legal compliance flags

### Using as a Claude Skill

Download the `.skill` file from [Releases](../../releases) and install it in your Claude environment. Then ask Claude to draft any physical security policy and the skill will activate automatically.

### Using as a Standalone Reference

Each file in `references/` is a self-contained template covering:

- **Required inputs** -- what you need to know before drafting
- **Key provisions** -- the substantive content sections your policy should include
- **ISO control mapping** -- which controls map to this policy domain
- **Compliance metrics** -- measurable KPIs for each domain
- **Legal considerations** -- jurisdiction-specific flags to review with counsel

You can use these reference files with any drafting process, not just Claude.

## Document Structure

Every policy generated by this framework follows this structure:

```
DOCUMENT CONTROL (title, ID, version, classification, owner, approver)
REVISION HISTORY

1. Purpose
2. Scope
3. Definitions
4. Roles and Responsibilities
5. Policy Statements / Procedure Steps
6. Compliance and Enforcement
7. Related Documents
8. Appendices
   - Appendix A: ISO Control Mapping
   - Appendix B: Legal Compliance Considerations
```

## Language Convention

The framework enforces RFC 2119-style language:

- **Shall** -- mandatory requirement
- **Should** -- recommended practice
- **May** -- optional or permissive

No aspirational language. No "try to" or "make an effort to." Every statement is enforceable or it does not belong in the policy.

## Limitations

- This framework provides templates and structure, not legal advice. Have your legal counsel review all policies before adoption.
- Regulatory requirements vary by jurisdiction, industry, and collective bargaining agreements. The legal flags in each reference file are starting points, not comprehensive legal analysis.
- The ISO mappings reference ISO 27001:2022 and ISO 22301:2019. Verify alignment with the specific version your organization is certifying against.
